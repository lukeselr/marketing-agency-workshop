# Phase 6.5 — Landing-page CRO

Generates a high-converting landing page per active campaign + A/B test infra.

## How it runs

```bash
bash ~/.claude/skills/marketing-agency/scripts/build-landing.py <campaign-slug>
```

## What it does

1. Detects site host from `.state/scrape/dns-host.json` (Framer / GHL / Vercel / Netlify / Shopify / Squarespace / unknown)
2. Picks matching builder skill: `framer-builder` / `ghl-landing-pages` / `vercel-deployment` / etc
3. Generates voice-graded copy via `delegate-creative` agent (3 headlines × 2 heroes × CTA × FAQ)
4. Injects Microsoft Clarity (free heatmap), Meta Pixel, LinkedIn Insight, GA4
5. Runs PageSpeed Insights API check on the current site
6. Drops `.state/landing/<campaign>/index.html` as DRAFT for owner approval

## Components per page

- Hero (3 headline variants × 2 hero variants for split test)
- Social proof strip (testimonials / stats)
- 3 value props
- FAQ for objection handling
- Sticky CTA
- Mobile-first, accessibility-clean (semantic HTML, ARIA labels)

## Tracking auto-injected

- Meta Pixel + CAPI dedup
- LinkedIn Insight Tag
- Google Ads conversion (EC4W)
- **Microsoft Clarity** heatmap — free, no rate limits
- GA4 events (page_view + form_submit)

## Owner approval gate

Page lands as DRAFT in the host platform. Owner reviews live preview, approves activation explicitly. Never auto-publish.
